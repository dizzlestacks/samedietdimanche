### How updates reach production
1. Agent edits files in Replit
2. Agent sends the user **only the changed files** (not the whole project zip — user has lost work before by replacing the whole folder)
3. User opens their **local GitHub Desktop YARDEES folder** on Mac
4. User **overwrites** the matching files with the new ones (drag/drop, replace when prompted)
5. GitHub Desktop shows the changes → user commits → pushes → Railway auto-deploys

## Overview
YARDEES is a marketplace web application for yard sales and thrift shopping, enabling users to post and browse listings categorized by location and item type. It supports both individual sellers and professional shops, fostering a community around second-hand goods with features like secure messaging, buyer verification, listing promotion, and interactive mapping. The platform aims to be the leading local commerce solution for second-hand goods, promoting community engagement and sustainable shopping.

## System Architecture
The application features a modern web stack: **React + Vite** for the frontend, styled with **TailwindCSS** and **shadcn/ui**, and animated with **Framer Motion**. The backend uses **Express.js**, **Drizzle ORM**, and **PostgreSQL**. Authentication is managed through **Replit Auth (OIDC)** and a custom email/password system with **passport-local** and **bcrypt**.

**UI/UX Decisions:**
- **Color Scheme:** Logo-matching green primary (HSL 130 65% 34%), warm gold accent (HSL 40 65% 50%), neutral warm backgrounds. Luxury-oriented with subdued saturation and refined shadows. Listing cards are minimal — no category/type/boost badges, just image, title, price, description, and location.
- **Typography:** Outfit (display, -0.02em tracking), DM Sans (body, -0.01em tracking, 1.65 line-height). Refined border radius (0.625rem).
- **Dark Mode:** Implemented with deep teal-green tones and localStorage persistence.
- **Animations & Effects:** Animated gradient text on hero title, floating decorative blur orbs, glass-morphism search card (`glass-card` utility), spring-physics hover/tap on listing cards and filter chips, staggered card entrance animations (first 8 only for performance), scroll-triggered section headers. Reduced-motion media query and a11y toggle disable all decorative animations. CSS utilities in `index.css`: `animate-gradient`, `animate-float`, `animate-float-slow`, `animate-pulse-glow`, `gradient-text`, `glass-card`, `hero-glow`, `stagger-item`. Card shells: `listing-card-shell`, `listing-card-{type}`, `listing-card-boosted`, `spotlight-card-shell`, `spotlight-shimmer`, `featured-category-card`.
- **Responsiveness:** Mobile-optimized, adapting grid layouts for different screen sizes.
- **Accessibility Widget:** Floating button with font size (Normal/Large/XL), high contrast, and reduce motion toggles. Persists in localStorage. CSS classes: `a11y-font-large`, `a11y-font-xl`, `a11y-high-contrast`, `a11y-reduce-motion`. i18n keys in all 6 locale files.

**Technical Implementations:**
- **State Management:** TanStack Query v5 for data fetching and caching.
- **Routing:** `wouter` for frontend, Express for backend API routes.
- **File Uploads:** `multer` with `sharp` for image optimization and local storage.
- **Mapping:** `Leaflet` and `react-leaflet` for interactive OpenStreetMap integration.
- **Charts:** `Recharts` for seller analytics.
- **Listing Management:** Supports various listing types, detailed fields, and up to 20 photos.
- **Photo Gallery:** Enhanced lightbox with zoom, pan, swipe navigation, and accessibility features.
- **User Profiles:** Editable profiles with bio, phone, website, location (city/country), favorite categories (up to 5), profile photo upload, and verification badge display. Member-since date shown. Phone is private (not exposed on public seller profile). Profile fields are validated server-side with length limits and category whitelist.
- **User Engagement:** Favorites, saved searches, buyer verification, and a comprehensive messaging system.
- **Real-time Chat:** WebSocket server (`server/websocket.ts`) at `/ws` with session-based auth, per-user socket tracking, typing indicators, online/offline presence (scoped to conversation partners), heartbeat ping/pong, and exponential-backoff reconnection. Client hook: `client/src/hooks/use-websocket.ts`. Messages page uses WebSocket for instant message delivery, typing dots, and online badges, with HTTP polling as fallback. Messages page has integrated Notifications tab (alongside Messages and ID Review) so users see offers, order updates, and all notifications in one place. Navbar notification dropdown includes "View all" link to `/messages?tab=notifications`. Bottom tab bar and navbar message icon show combined unread count (messages + notifications).
- **Event Management:** Yard sale event creation with RSVP and calendar synchronization.
- **Offers & Orders:** Full purchase flow — offers, counter-offers, Stripe Checkout payment, order management with tracking. When a seller accepts an offer, an order is created with 5% platform fee. Buyer pays via Stripe Checkout, funds held in escrow until delivery confirmed. Status timeline: pending → paid (escrow held) → shipped → delivered (escrow released). `OrderPaymentSuccess` page handles post-payment confirmation.
- **Escrow System:** Full buyer & seller protection. Payments held in escrow with atomic state transitions (conditional WHERE clauses prevent race conditions). Buyers confirm delivery to release funds; 14-day auto-release protects sellers if buyer doesn't respond. Unpaid orders auto-cancel after 4 days with listing relisted. Both parties can file disputes (freezes funds). Escrow guarantee banner on Orders page. Protection messaging in offer dialog. Dispute notifications to both parties with email alerts. State machine: only `shipped + held` → `delivered + released`; only `held` → `disputed`. Payment confirmation is atomic (`WHERE status='pending'`) to prevent race conditions with auto-cancel.
- **Referral Program:** Incentivizes user growth with unique referral codes and boost credits.
- **Admin Panel:** Standalone HTML page for managing users, listings, verifications, disputes, and analytics, secured by database-persisted bcrypt-hashed password and token-based authentication. First access has no password (open entry); once inside, a security banner prompts to set a password which persists across server restarts via the `admin_settings` table. Disputes tab shows all disputed orders with buyer/seller info, amounts, and dispute reasons. Admins can release funds to seller or refund buyer with optional admin notes. Both parties receive notifications and emails when disputes are resolved. Admin receives in-app notification when new disputes are filed.
- **Dispute Chat:** Three-way messaging system for disputed orders. `dispute_messages` table stores messages with `senderRole` (buyer/seller/admin). Buyer and seller see a collapsible "Dispute Discussion" panel on their order card (Orders page). Admin panel has an integrated chat in each dispute card. All three parties receive in-app notifications when a new message is posted. Auto-refreshes every 5 seconds when open. API: `GET/POST /api/orders/:id/dispute-chat` (supports both session auth and admin bearer token auth).
- **PWA (Progressive Web App):** Full PWA setup with web app manifest, service worker (v3) for offline caching with separate static/API/image caches, stale-while-revalidate for uploaded images, app icons, install prompt, and safe-area inset handling. Offline browsing shows cached listings with full filter/sort support (search, category, price, condition, location, listing type). `OfflineBanner` shows cached listing count and freshness. IndexedDB via `offlineStorage.ts` for persistent listing cache. `applyOfflineFilters()` in `use-listings.ts` handles offline search/filter/sort parity.
- **Explore Page:** Dedicated `/explore` page accessible via bottom tab bar and navbar. Features a full category browsing grid (all 15 categories with icons, colors, and item counts), nearby shops section (Google Places), and recently added listings. Links to `/category/:name` for deep browsing.
- **Price Drop Alerts:** Users get notified when a favorited listing's price drops.
- **Smart Price Suggestions:** API returns avg/min/max prices from sold listings by category. `PriceSuggestion` component in ListingForm.
- **Social Sharing Cards:** Server-side OG meta tag generation with image, title, price, location. Twitter card support.
- **Seasonal Collections:** `collections` table, 8 seeded collections, `/api/collections` endpoints, `/collections` and `/collections/:slug` pages. Collections section visible on Homepage and Explore page, linked from Navbar.
- **Video Listings:** `/api/upload/video` endpoint (50MB), `videos TEXT[]` on listings, video player in ListingDetail, max 3 per listing.
- **Seller Storefront:** `storefront_bio`, `storefront_banner`, `storefront_tagline` on users table. Editable via Dashboard profile form (tagline, bio, banner URL fields). `/api/profile/storefront` PATCH, `/api/users/:id/storefront` GET. Enhanced SellerProfile with banner, bio, tagline, member-since. Storefront fields also saved through main PATCH `/api/profile`.
- **Wishlist Matching:** `wishlists` table with keywords, category, max_price. CRUD at `/api/wishlists`. Auto-matching on new listing creation sends notifications.
- **Barcode/ISBN Scanner:** `/api/barcode/:code` looks up Open Food Facts + Open Library. Camera barcode detection via BarcodeDetector API. Pre-fills listing form via URL params.
- **Loyalty/Rewards Program:** `loyalty_points` and `loyalty_transactions` tables. Daily login bonus, points for listings/sales/reviews/referrals. Tier system (bronze/silver/gold/platinum). `/rewards` page. `awardLoyaltyPoints()` helper in routes.ts awards points on listing_created, listing_sold, review_written with deduplication keys to prevent farming.
- **Auction/Bidding:** `auctions` and `bids` tables. Create auctions on listings, place bids with minimum increment, outbid notifications, bid history. `/auctions` page.
- **Neighborhood Yard Sale Events:** `neighborhood_events` and `neighborhood_participants` tables. Create/join neighborhood events with location, dates, participant limits. `/neighborhood-events` page.
- **Delivery/Shipping Options:** `shipping_options` table per listing. Methods, prices, estimated days, free shipping flag. API at `/api/listings/:id/shipping`. `ShippingManager` component in ListingForm for adding shipping options when editing listings. Shipping options displayed on ListingDetail page.
- **Yard Sale Date Picker:** `saleStartDate` and `saleEndDate` timestamp columns on listings table. ListingForm shows `datetime-local` inputs for yard sale type in the "When & Where" card. On create/update, if dates are set, an event is auto-created/updated in the `events` table. Cleanup scheduler auto-deactivates (sets `isSold=true`) yard sales whose `saleEndDate` has passed, with a notification sent to the seller. ListingDetail shows an orange "Sale Dates" card for yard sales with formatted start/end dates.
- **Wallet & Payouts:** `payouts` table tracks withdrawal requests. Sellers see `/wallet` page with available balance (total released escrow minus completed + pending withdrawals), total earnings, total withdrawn. Withdrawal form supports three payout methods: Bank Transfer (ACH), Wire Transfer, and PayPal (email). `paypalEmail` column on payouts table. PayPal withdrawals require only email; bank methods require account holder, last 4 digits, routing number. $5 minimum. Admin panel Payouts tab shows PayPal email or bank details depending on method. Both parties get notifications on status changes. Linked from Dashboard quick-action grid.
- **Community Reviews & Tips:** `community_tips` and `tip_votes` tables. Create tips, upvote/downvote, category filtering, featured tips. `/tips` page.
- **Core Features:** Email verification, password reset, advanced search/filtering (including radius/distance filter using Haversine formula on listing lat/lng), seller tools (listing boosting via Stripe, analytics), nearby thrift shops section on homepage (Google Places API, shown when no filters active or Stores tab active), community features (interactive maps, reporting), bulk CSV import, SEO optimization (OG meta tags, sitemap, robots.txt, JSON-LD), in-app notifications, search autocomplete, photo drag-and-drop, admin report responses, seller review replies, comprehensive email notifications via Resend, content moderation, legal pages, branded 404, animated loading states, feedback system, onboarding tour (`driver.js`), mobile bottom tab bar, pull-to-refresh, haptic feedback, and native touch feedback.
